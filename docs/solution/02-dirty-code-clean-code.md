# Clean code VS Dirty code

## Qu'est-ce que du code sale ?

- Code redondant
- Code qui ne suit pas de règles/normes de naming  
- Fonctions multifonctionnelles avec un scope trop large
- Fonction trop longues
- Fausse documentation (définition creuses de paramètres de fonctions...)
- print à la place du logging 

## Qu'est-ce que du code propre ?

- Bonnes dépendances (bien séparées)
- Suivre les principes SOLID
- Code extensible
- Réduire le couplage
- Favoriser la cohésion du code 
- Couche d'abstraction pour diminuer les dépendances sur l'implémentation concrète.
- Code bien documenté et commenté
- Explication claire du rôle de chaque élément
- Code lisible par tous
